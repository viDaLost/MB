# Ночной город / Последний бункер

Mobile-first PWA с двумя полноценными играми ведущего для компании на одном телефоне. Приложение работает без сервера, сохраняет текущую партию локально и не отправляет личные данные игроков в сеть.

## Что входит

### Ночной город — «Мафия»

- 5–16 игроков и автоматический баланс состава.
- Три набора правил: классика, нуар с ночным поиском Комиссара Доном и хаос с Маньяком.
- Безопасная раздача через удержание экрана; мафия видит союзников только в своей карточке.
- Последовательная ночь: синдикат, Доктор, Комиссар и Маньяк.
- Корректная защита Доктора, одновременные атаки и приватная проверка Комиссара.
- Таймер обсуждения, несколько номинаций, бюджет голосов, ничья и день без исключения.
- Автоматические условия победы города, синдиката или одиночной роли.
- Публичный журнал, состав, одношаговая отмена итогового действия и финальное раскрытие ролей.

### Последний бункер

- 4–16 кандидатов, 2–10 мест и три тона сценария: реализм, кино, абсурд.
- Детерминированный сценарий по seed: катастрофа, срок изоляции, площадь, прочность, комнаты, ограничение и релевантные запасы.
- Связные досье без невозможных сочетаний возраста, профессии, пола и здоровья.
- Уникальные профессии, увлечения, фобии, черты и предметы для максимального стола.
- Одна приватная переброска свойства до начала дебатов.
- Безопасная повторная проверка личного досье и карты действия через удержание экрана.
- Поэтапное раскрытие данных, таймер речи, карты действий, голосования и право вето.
- Кризисы между раундами: выбор подхода проверяет реальные компетенции группы и меняет воздух, воду, еду, энергию и мораль.
- Автоматический финал при достижении вместимости, оценка устойчивости команды и эпилог.

## Приватность и надёжность

- Скрытые карточки открываются только после удержания в течение 650 мс.
- При сворачивании вкладки карточка автоматически закрывается.
- Открытая роль, анкета и результат проверки Комиссара никогда не восстанавливаются открытыми после перезагрузки.
- Сохранения проверяются схемами Zod. Повреждённая или устаревшая запись безопасно удаляется вместо падения приложения.
- Нет нативных `alert`/`confirm`: все решения оформлены доступными нижними панелями.
- Учитываются safe-area, клавиатурная навигация, фокус, `prefers-reduced-motion` и сенсорные цели не меньше 42–48 px.

## Технологии

- React 19 + TypeScript 6 + Vite 8.
- Чистые reducer-движки игр, отделённые от UI и контента.
- Seeded RNG для воспроизводимых партий.
- Zod для версионированных локальных сохранений.
- Vite PWA + Workbox для установки и офлайн-кэша.
- Разделение кода: каждая игра загружается отдельным чанком.
- Vitest + Testing Library и Playwright для мобильных сценариев.

## Запуск

Требуется Node.js 20.19+ или 22.12+.

```bash
npm install
npm run dev
```

Сборка для публикации:

```bash
npm run build
npm run preview
```

Для установки PWA production-сборка должна открываться по HTTPS (исключение браузеров — `localhost`).

## Публикация на GitHub Pages

Проект уже содержит workflow `.github/workflows/deploy-pages.yml`. Он проверяет код, собирает приложение и публикует папку `dist` после каждого push в основную ветку репозитория — независимо от её имени.

1. Распакуйте архив в корень репозитория и отправьте файлы на GitHub.
2. Откройте **Settings → Pages**.
3. В разделе **Build and deployment → Source** выберите **GitHub Actions**.
4. Запустите workflow вручную во вкладке **Actions** или сделайте push в основную ветку.

Путь публикации определяется автоматически:

- `username.github.io/repository/` — используется `/repository/`;
- `username.github.io/` и пользовательский домен — используется `/`.

Поэтому ассеты, маршрутизация, PWA-манифест и офлайн-режим работают и в корне домена, и во вложенной папке репозитория. Для ручной сборки с собственным путём можно передать переменную:

```bash
VITE_BASE_PATH=/my-path/ npm run build
```

Не публикуйте исходники через режим **Deploy from a branch**: приложению нужен этап сборки Vite, который уже настроен в GitHub Actions.

## Проверки

```bash
npm run lint       # статический анализ
npm test           # логика, сохранения и доступные компоненты
npm run build      # TypeScript + production PWA
npx playwright install chromium
npm run test:e2e   # Pixel 5 и компактный экран 320×568
npm run check      # lint + unit + build
```

## Структура

```text
src/
  components/       общие доступные UI-компоненты
  data/             роли, катастрофы, профессии, кризисы и контент
  games/mafia/      движок, тесты и интерфейс «Мафии»
  games/bunker/     движок, тесты и интерфейс «Бункера»
  lib/              RNG, сохранения, обратная связь и ключи сессий
  App.tsx            маршрутизация, главная и настройки
public/              фоновые изображения и PWA-иконки
e2e/                 сквозные мобильные сценарии
```

Игры не требуют регистрации, базы данных, ведущего-сервера или подключения к интернету.
